# Dotfiles — ~/.config

Arch/CachyOS Linux dotfiles for a Hyprland desktop. Configs live directly in `~/.config/` as a git repo; `install.sh` symlinks each managed directory into place on a fresh machine.

## Install

```bash
# Fresh machine setup (Arch only)
bash install.sh
```

Managed directories: `hypr kitty waybar theme rofi wallust gtk-3.0 gtk-4.0 zsh`
Managed files: `pavucontrol.ini QtProject.conf`

Backs up any existing configs to `~/.config_backup_<timestamp>/` before symlinking.

## Architecture

### Hyprland (`hypr/`)

Entry point is `hypr/hyprland.conf`, which sources all sub-configs:

| File | Purpose |
|------|---------|
| `config/autostart.conf` | `exec-once` entries (waybar, awww, mako, etc.) |
| `config/keybinds.conf` | All keybinds — sources `defaults.conf` for `$mainMod` and app vars |
| `config/defaults.conf` | Variables: `$mainMod`, `$terminal`, `$browser`, `$menu`, etc. |
| `config/windowrules.conf` | Window rules, transparency, floats, gaming suppress, workspace gaps |
| `config/animations.conf` | Animation config |
| `config/decorations.conf` | Borders, blur, shadows |
| `config/input.conf` | Keyboard/mouse/touchpad |
| `config/variables.conf` | Hyprland general/layout variables |
| `config/enviornment.conf` | `env` exports (note: intentional filename typo — matches the `source =` line) |
| `monitors.conf` | Monitor layout — **auto-generated by nwg-displays, do not hand-edit** |

### Named Workspaces

Workspaces 6–9 have names and dedicated keybinds:

| Key | Workspace | Name |
|-----|-----------|------|
| `Super+D` | 6 | dev |
| `Super+S` | 7 | server |
| `Super+W` | 8 | work |
| `Super+G` | 9 | game |
| `Super+C` | named | config |
| `Super+K` / `Super+=` | special | magic (scratchpad) |
| `Super+Tab` | — | cycle monitor focus |

### Wallpaper & Theming (`wallust/`)

wallust generates color schemes from the current wallpaper and distributes them to all apps:

```bash
# Apply a new wallpaper + regenerate theme
wallust run ~/Pictures/Wallpapers/generated/wallpaper.png
```

Templates in `wallust/templates/` generate output to:
- `~/.config/waybar/colors-waybar.css`
- `~/.config/rofi/themes/wallust-colors.rasi`
- `~/.config/gtk-3.0/wallust-colors.css` and `gtk-4.0/`
- `~/.config/yazi/theme.toml`
- `~/.cache/wallust/` (kitty, hypr colors, vscode theme)

The `reload_ui` hook auto-reloads hyprland and waybar after generation.

### Wallpaper Daemon

Uses **awww** (not swww or hyprpaper). Daemon started via autostart:
```bash
exec-once = awww-daemon
exec-once = sh -lc 'sleep 1; awww img $HOME/Pictures/Wallpapers/generated/wallpaper.png --resize crop --transition-type none'
```

### Notifications

Uses **swaync**. Waybar shows the notification bell via `custom/swaync` (uses `swaync-client`). No custom swaync config in this repo — running with swaync defaults.

### Zsh (`zsh/`)

Config lives at `~/.config/zsh/` with `ZDOTDIR` set via `~/.zshenv`. Stack:
- Oh My Zsh + Powerlevel10k theme
- Plugins: `git zsh-autosuggestions zsh-syntax-highlighting`
- `colorls` for `ls` / `lc` aliases (requires ruby gem)

## Monitor Layout

4-monitor setup managed by nwg-displays:

| Output | Resolution | Position |
|--------|-----------|---------|
| DP-3 | 2560×1440 @ 144Hz | left (0,0) |
| DP-2 | 2560×1440 @ 144Hz | center (2560,0) |
| HDMI-A-1 | 1920×1080 @ 60Hz | below center (1636,1440) |
| DP-4 | 1920×1080 @ 60Hz | right (7040,0) |

To update layout, use nwg-displays — it rewrites `monitors.conf` automatically.

## Gotchas

- **`monitors.conf`** is owned by nwg-displays and gets overwritten on each display reconfigure. The `.bak` is also untracked — add to `.gitignore` or delete.
- **`hypr/workspaces.conf`** is an empty untracked file — placeholder, not yet sourced in `hyprland.conf`.
- **`packages.txt`** lists `hyprpaper` and `waypaper` but the active wallpaper tool is `awww`. These entries are stale.
- **`hypr/hyprpaper.conf`** still exists in the repo but is unused.
- **wallust-generated files** (`colors-waybar.css`, `colors.css`, `wallust-colors.rasi`, etc.) are live output — don't manually edit them.
- **`theme/`** has `apply-discord-theme.sh` and `apply-vscode-theme.sh` called by wallust hooks.
